# Review Sentiment Analysis

Веб-приложение для импорта отзывов о фильмах по ссылке IMDb, загрузки отзывов из TMDB, сохранения их в базу данных и оценки тональности с помощью обученной модели машинного обучения.

## О проекте

Проект решает задачу автоматического анализа тональности пользовательских отзывов о фильмах.

Пользователь вставляет ссылку на фильм с IMDb, после чего приложение:

1. извлекает `IMDb ID` из ссылки;
2. ищет соответствующий фильм в TMDB;
3. загружает доступные отзывы к фильму через TMDB API;
4. отбирает только отзывы с выраженной оценкой:
   - `rating <= 4` — негативный отзыв;
   - `rating >= 7` — позитивный отзыв;
   - отзывы с нейтральными оценками пропускаются;
5. сохраняет отзывы в PostgreSQL;
6. прогоняет тексты через локально сохранённую модель классификации;
7. показывает сравнение:
   - «реальной» тональности по пользовательскому рейтингу;
   - тональности, предсказанной моделью.

Таким образом, проект объединяет **сбор данных**, **хранение**, **ML-инференс** и **веб-интерфейс** в одном приложении.

### Что находится в папках

#### `app/`
Основная логика веб-приложения.

- `main.py` — FastAPI-приложение и маршруты
- `db.py` — подключение к БД и создание сессий SQLAlchemy
- `models.py` — ORM-модели `Movie` и `Review`
- `tmdb.py` — работа с TMDB API
- `status_checker.py` — подсчёт тональности по рейтингу и по ML-модели
- `templates/` — HTML-шаблоны интерфейса

#### `best_model/`
Сохранённая обученная модель для классификации отзывов.

#### `pipeline/`
Материалы для обучения и экспериментов с моделями.

- `data_download.py` — загрузка датасета IMDb Reviews через KaggleHub
- `data_processing.ipynb` — построение базовых моделей (TF-IDF + LinearSVC / LogisticRegression, а также эксперименты с CNN)
- `model_finetuning.ipynb` — дообучение трансформерной модели на отзывах, сохранённых из TMDB

## Как это работает

### 1. Импорт фильма
Пользователь отправляет IMDb-ссылку через форму на главной странице.

### 2. Поиск фильма
Приложение извлекает идентификатор вида `tt1234567` и ищет фильм в TMDB через endpoint `find/{imdb_id}`.

### 3. Загрузка отзывов
После нахождения фильма приложение получает все страницы отзывов через TMDB API.

### 4. Фильтрация отзывов
Сохраняются только отзывы, где присутствует рейтинг и он явно указывает на позитивную или негативную тональность.

### 5. Сохранение в БД
Информация о фильме и отзывы сохраняются в PostgreSQL. Повторные отзывы не добавляются.

### 6. Оценка тональности
Для всех сохранённых отзывов считаются:

- **original sentiment** — на основе рейтинга пользователя;
- **model sentiment** — на основе предсказания обученной модели.

### 7. Отображение результата
Пользователь видит количество:

- новых сохранённых отзывов;
- всех отзывов в базе;
- позитивных и негативных отзывов по рейтингу;
- позитивных и негативных отзывов по модели.

## Обучение модели

В проекте присутствуют два ML-направления:

### 1. Базовые модели
В ноутбуке `pipeline/data_processing.ipynb` проводятся эксперименты с классическими подходами:

- `TF-IDF + LinearSVC`
- `TF-IDF + LogisticRegression`

Лучшая базовая модель показала примерно:

- `CV accuracy ≈ 0.8822`
- `Test accuracy ≈ 0.9019`

### 2. Дообучение трансформера
В `pipeline/model_finetuning.ipynb` выполняется дообучение модели на отзывах, собранных из TMDB и сохранённых в базе.

Для обучения используются:

- локально сохранённая модель из `best_model/`
- датасет из TMDB-отзывов
- тестовый IMDb-датасет из JSON

Итоговая модель используется приложением для инференса в рантайме.

## Модели данных

### `Movie`
Сущность фильма содержит:

- `id`
- `imdb_id`
- `tmdb_id`
- `title`
- `imdb_url`

### `Review`
Сущность отзыва содержит:

- `id`
- `movie_id`
- `tmdb_review_id`
- `author`
- `author_username`
- `rating`
- `content`
- `review_url`
- `created_at`
- `updated_at`

Для `tmdb_review_id` задано ограничение уникальности, чтобы не сохранять дубликаты.

## Установка и запуск

### 1. Клонирование репозитория
```bash
git clone https://github.com/fert1414/review-sentiment-analysis.git
cd review-sentiment-analysis
```

### 2. Создание виртуального окружения
```bash
python -m venv venv
source venv/bin/activate
```

Для Windows:
```bash
venv\Scripts\activate
```

### 3. Установка зависимостей
```bash
pip install -r requirements.txt
```

### 4. Настройка переменных окружения
Файл `.env` в корне проекта:

```env
DATABASE_URL=postgresql://postgres:password@localhost:5432/review_sentiment
TMDB_ACCESS_TOKEN=your_tmdb_access_token
```

### 5. Запуск приложения
```bash
uvicorn app.main:app --reload
```

После запуска приложение будет доступно по адресу:

```text
http://127.0.0.1:8000
```
